# 报表系统快速使用指南

## 🚀 快速开始

### 1. 启动系统

#### 后端
```bash
# 确保已安装依赖
pip install openpyxl

# 启动后端服务
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

#### 前端
```bash
cd web-admin

# 安装依赖（首次运行）
npm install

# 启动开发服务器
npm run dev
```

### 2. 登录系统
1. 访问 `http://localhost:5173`（或前端配置的端口）
2. 使用管理员账号登录
3. 需要以下角色之一：
   - `super_admin`: 超级管理员
   - `event_admin`: 活动管理员
   - `reviewer`: 审核员

### 3. 访问报表中心
点击侧边栏的"报表中心"菜单，展开后可以看到6个子菜单：
- 📊 总览看板
- 🏪 摊位报表
- 🏆 摊位排行榜
- 🏆 商品排行榜
- 🔍 异常审计
- 📥 报表导出

## 📊 功能使用指南

### 总览看板

**路径**: 报表中心 → 总览看板

**功能**:
- 查看8个关键指标卡片
- 按活动筛选数据
- 导出总览统计 Excel

**操作步骤**:
1. 进入总览看板页面
2. （可选）选择特定活动，不选则显示所有活动数据
3. 查看各项统计指标
4. 点击"导出 Excel"按钮下载报表

**指标说明**:
- **总发放额度**: 所有 issue 类型交易的总额
- **总充值额**: 所有 recharge 类型交易的总额
- **总消费额**: 所有 pay 类型交易的总额
- **总退款额**: 所有 refund 类型交易的总额
- **净消费额**: 总消费 - 总退款
- **总交易笔数**: 所有交易记录数
- **参与者数量**: 参与活动的人数
- **摊位数量**: 活动中的摊位数

---

### 摊位报表

**路径**: 报表中心 → 摊位报表

**功能**:
- 查看所有摊位的经营数据
- 多列排序
- 按活动筛选
- 导出摊位报表 Excel

**操作步骤**:
1. 进入摊位报表页面
2. （可选）选择特定活动
3. 查看表格数据
4. 点击列标题进行排序
5. 点击"导出 Excel"按钮下载报表

**数据列说明**:
- **摊位ID**: 摊位唯一标识
- **摊位名称**: 摊位的名称
- **班级名称**: 运营摊位的班级
- **营业额**: pay 类型交易总额
- **退款额**: refund 类型交易总额
- **净收入**: 营业额 - 退款额
- **销量**: pay 类型交易笔数
- **总成本**: 基于商品成本价计算
- **利润**: 净收入 - 总成本
- **利润率**: (利润 / 净收入) × 100%

**利润率颜色说明**:
- 🟢 绿色: ≥ 50%（优秀）
- 🔵 蓝色: 30% - 50%（良好）
- 🟡 黄色: 0% - 30%（一般）
- 🔴 红色: < 0%（亏损）

---

### 摊位排行榜

**路径**: 报表中心 → 摊位排行榜

**功能**:
- 查看摊位营业额、利润、利润率排行
- 前三名特殊显示
- 支持 TOP 10/20/50 切换

**操作步骤**:
1. 进入摊位排行榜页面
2. （可选）选择特定活动
3. 选择排行榜类型：
   - 营业额 TOP
   - 利润 TOP
   - 利润率 TOP
4. 选择显示数量（TOP 10/20/50）
5. 查看排行榜

**排名显示**:
- 🥇 第1名: 金色皇冠图标 + 黄色背景
- 🥈 第2名: 银色奖杯图标 + 蓝色背景
- 🥉 第3名: 铜色奖杯图标 + 橙色背景
- 其他: 数字排名

---

### 商品排行榜

**路径**: 报表中心 → 商品排行榜

**功能**:
- 查看商品销量、收入、利润排行
- 前三名特殊显示
- 支持 TOP 10/20/50 切换

**操作步骤**:
1. 进入商品排行榜页面
2. （可选）选择特定活动
3. 选择排序指标：
   - 销量（件数）
   - 收入（元）
   - 利润（元）
4. 选择显示数量（TOP 10/20/50）
5. 查看排行榜

**使用场景**:
- 发现爆款商品
- 分析商品盈利能力
- 优化商品定价策略
- 调整商品库存

---

### 异常审计

**路径**: 报表中心 → 异常审计

**功能**:
- 检测和展示异常交易
- 按异常类型筛选
- 查看详细交易信息

**操作步骤**:
1. 进入异常审计页面
2. （可选）选择特定活动
3. 选择异常类型：
   - 全部异常
   - 高频退款
   - 大额更正
   - 可疑操作
4. 选择显示数量（50/100/200/500条）
5. 查看异常记录

**异常类型说明**:

#### 🟡 高频退款
- **检测规则**: 同一操作员在1小时内退款超过5次
- **风险**: 可能存在操作失误或恶意退款
- **建议**: 核实退款原因，必要时联系操作员

#### 🔴 大额更正
- **检测规则**: adjust 类型交易且金额 > 1000元
- **风险**: 大额调整可能影响账目准确性
- **建议**: 核实调整原因和审批流程

#### 🟣 可疑操作
- **检测规则**: 深夜时段（22:00-06:00）的交易
- **风险**: 非正常营业时间的交易可能存在异常
- **建议**: 核实交易真实性和必要性

### 报表导出

**路径**: 报表中心 → 报表导出

**功能**:
- 统一的报表导出入口
- 支持4种报表类型
- 自动生成文件名

**操作步骤**:
1. 进入报表导出页面
2. （可选）选择特定活动
3. 选择要导出的报表类型：
   - 总览统计报表
   - 摊位报表
   - 商品报表
   - 交易流水报表
4. 点击"导出 Excel"按钮
5. 浏览器自动下载文件

**文件命名规则**:
```
{报表类型}_{活动名称}_{日期}.xlsx

示例:
- 总览统计报表_春季校园节_2024-05-02.xlsx
- 摊位报表_全部活动_2024-05-02.xlsx
- 商品报表_美食节_2024-05-02.xlsx
```

**Excel 文件特点**:
- ✅ 包含完整的表头和数据
- ✅ 自动格式化（颜色、对齐、边框）
- ✅ 金额保留两位小数
- ✅ 支持中文文件名
- ✅ 兼容 Microsoft Excel 和 WPS

**交易流水导出限制**:
- 最多导出最近 10000 条记录
- 如需完整数据，建议分批导出或联系管理员

## 🎯 常见使用场景

### 场景1: 活动结束后的总结报告
1. 进入"总览看板"，选择对应活动
2. 截图或导出总览统计数据
3. 进入"摊位报表"，导出详细数据
4. 进入"摊位排行榜"，查看表现优秀的摊位
5. 进入"商品排行榜"，分析热销商品
6. 整理数据，生成活动总结报告

### 场景2: 实时监控异常交易
1. 进入"异常审计"页面
2. 选择"全部异常"
3. 定期刷新页面（建议每小时一次）
4. 发现异常后，记录交易ID
5. 联系相关操作员核实情况
6. 必要时进行账目调整

### 场景3: 摊位经营分析
1. 进入"摊位报表"页面
2. 选择对应活动
3. 按"利润率"列降序排序
4. 分析高利润率摊位的经营策略
5. 对比低利润率摊位，找出问题
6. 提供改进建议

### 场景4: 商品定价优化
1. 进入"商品排行榜"页面
2. 切换到"利润"排序
3. 找出高利润商品
4. 切换到"销量"排序
5. 找出高销量但低利润的商品
6. 建议调整定价或成本控制

### 场景5: 财务对账
1. 进入"报表导出"页面
2. 导出"交易流水报表"
3. 导出"摊位报表"
4. 使用 Excel 进行数据核对
5. 检查总额是否匹配
6. 发现差异时查看"异常审计"

## 💡 使用技巧

### 技巧1: 快速筛选活动
- 所有报表页面都支持按活动筛选
- 不选择活动时，显示所有活动的汇总数据
- 选择活动后，只显示该活动的数据

### 技巧2: 表格排序
- 点击列标题可以进行排序
- 再次点击切换升序/降序
- 支持多列排序（按住 Shift 键点击）

### 技巧3: 数据导出
- 每个报表页面都有独立的导出按钮
- 也可以在"报表导出"页面统一导出
- 导出的 Excel 文件可以直接用于打印或分享

### 技巧4: 异常监控
- 建议每天至少检查一次"异常审计"
- 重点关注"高频退款"和"大额更正"
- 发现异常及时处理，避免问题扩大

### 技巧5: 数据分析
- 结合多个报表进行综合分析
- 使用排行榜发现优秀案例
- 使用摊位报表进行横向对比
- 使用商品报表进行纵向分析

## ❓ 常见问题

### Q1: 为什么有些摊位的利润率显示为"-"？
**A**: 利润率计算需要净收入大于0。如果摊位没有销售记录或净收入为0，则利润率显示为"-"。

### Q2: 为什么商品的利润为0？
**A**: 利润计算依赖商品的成本价。如果商品没有设置成本价，则利润默认为0。请在"商品管理"中设置成本价。

### Q3: 导出的 Excel 文件打不开？
**A**: 请确保使用 Microsoft Excel 2010 或更高版本，或使用 WPS 表格。如果仍然无法打开，请联系技术支持。

### Q4: 为什么看不到某个活动的数据？
**A**: 请确认：
1. 该活动是否已创建
2. 该活动是否有交易记录
3. 您的账号是否有权限查看该活动

### Q5: 异常审计中的"可疑操作"是否一定有问题？
**A**: 不一定。深夜交易可能是正常的补录或调整操作。建议核实具体情况后再做判断。

### Q6: 如何导出所有活动的数据？
**A**: 在活动筛选器中不选择任何活动（或点击"清除"按钮），然后导出即可。

### Q7: 交易流水导出为什么只有10000条？
**A**: 为了保证导出性能和文件大小，限制了最多10000条。如需完整数据，请分批导出或联系管理员。

### Q8: 数据更新频率是多少？
**A**: 所有报表数据都是实时查询的，刷新页面即可看到最新数据。

## 🔐 权限说明

### 可访问报表功能的角色
- ✅ `super_admin`: 超级管理员（所有权限）
- ✅ `event_admin`: 活动管理员（可查看管理的活动）
- ✅ `reviewer`: 审核员（只读权限）

### 不可访问的角色
- ❌ `cashier`: 收银员（只能使用收银功能）
- ❌ `booth_admin`: 摊位管理员（只能管理自己的摊位）

### 权限不足时的处理
如果您看不到"报表中心"菜单或访问报表页面时提示权限不足，请联系系统管理员申请权限。
