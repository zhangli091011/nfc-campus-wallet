## 🚀 启动步骤

### 1. 安装依赖（如果还没安装）

```bash
cd web-admin
npm install
```

### 2. 启动后端服务

```bash
# 在项目根目录
python start_server.py
```

后端将运行在 `http://localhost:8000`

### 3. 启动前端服务

```bash
# 在 web-admin 目录
npm run dev
```

前端将运行在 `http://localhost:5173`

### 4. 访问系统

1. **登录管理后台**: `http://localhost:5173/login`
2. **在左侧菜单找到"股市大屏"菜单项**（在"报表中心"下方）
3. **点击菜单项**，大屏将在新窗口打开
4. **或直接访问**: `http://localhost:5173/stock-dashboard`

## 🔧 配置说明

### 环境变量

确保 `web-admin/.env` 文件包含：

```env
VITE_API_URL=http://localhost:8000
```

### API端点

大屏使用以下API：

- `GET /api/stock/stats/{event_id}` - 获取市场统计
- `GET /api/stock/settlement/event/{event_id}` - 获取结算数据

### 数据刷新

- **自动刷新**: 每5秒自动刷新数据
- **手动刷新**: 刷新浏览器页面

## 🐛 故障排查

### 问题1: 菜单项不显示

**解决方案**:
- 确认已登录系统
- 检查 `Layout/index.tsx` 中是否添加了菜单项
- 清除浏览器缓存并刷新

### 问题2: 点击菜单无反应

**解决方案**:
- 检查浏览器是否阻止了弹出窗口
- 在浏览器设置中允许弹出窗口
- 或直接访问 `http://localhost:5173/stock-dashboard`

### 问题3: 大屏显示空白

**解决方案**:
- 检查后端服务是否运行
- 检查浏览器控制台是否有错误
- 确认 API 地址配置正确
- 检查是否有数据（需要先进行股票交易）

### 问题4: 图表不显示

**解决方案**:
- 确认 echarts 依赖已安装: `npm list echarts`
- 如未安装: `npm install echarts`
- 检查是否有结算数据

### 问题5: 样式错误

**解决方案**:
- 确认 `StockDashboard.css` 文件存在
- 检查 CSS 文件是否正确导入
- 清除浏览器缓存

## 🔐 权限说明

### 访问权限

- **需要登录**: 是
- **角色要求**: 所有已登录用户
- **可配置**: 可在路由中添加角色验证

### 修改权限验证

如需限制只有管理员访问，修改 `routes/index.tsx`:

```typescript
const AdminRoute = ({ children }: { children: React.ReactNode }) => {
  const user = getUser()
  return user?.role === 'admin' ? <>{children}</> : <Navigate to="/dashboard" replace />
}

// 使用
<Route
  path="/stock-dashboard"
  element={
    <AdminRoute>
      <StockDashboard />
    </AdminRoute>
  }
/>
```
