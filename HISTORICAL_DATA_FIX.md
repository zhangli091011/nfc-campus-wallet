# 历史数据修复说明

## 问题描述
Web管理后台显示的历史交易金额不正确：
- 交易ID 2, 3: 显示 ¥1.11（应为 ¥111.00）
- 交易ID 5: 显示 ¥2.22（应为 ¥222.00）

## 原因分析
这些交易是在修复安卓端金额单位错误**之前**创建的，使用了错误的金额单位：
- 数据库中存储为 111 分（应为 11100 分）
- 导致显示为 ¥1.11（应为 ¥111.00）

## 修复过程

### 1. 检查问题
运行 `check_transaction_amounts.py` 发现3条错误记录：
```
ID    类型    金额(分)    金额(元)
2     pay     111         1.11      ❌ 应为 111.00
3     pay     111         1.11      ❌ 应为 111.00
5     pay     222         2.22      ❌ 应为 222.00
```

### 2. 执行修复
运行 `fix_historical_transaction_amounts.py`：
- 自动识别可疑交易（金额 < 1000分 且类型为支付）
- 将金额字段乘以 100
- 将余额字段乘以 100
- 提交数据库更改

### 3. 验证结果
修复后所有交易金额正确：
```
ID    类型    金额(分)    金额(元)
2     pay     11100       111.00    ✅
3     pay     11100       111.00    ✅
5     pay     22200       222.00    ✅
6     pay     11100       111.00    ✅
7     pay     11100       111.00    ✅
```

## 修复的文件

### 数据库表：transactions
修复了以下字段：
- `amount`: 交易金额（分）
- `balance_before`: 交易前余额（分）
- `balance_after`: 交易后余额（分）

### 受影响的交易
- 交易ID 2: 111 → 11100 分
- 交易ID 3: 111 → 11100 分
- 交易ID 5: 222 → 22200 分

## 验证步骤

### 1. Web管理后台
登录 Web 管理后台，检查：
- **首页**：最近交易应显示正确金额
- **交易历史页**：所有交易金额应正确
- **报表页**：统计数据应正确

### 2. 数据库查询
```sql
-- 查看所有交易记录
SELECT 
    id,
    type,
    amount AS amount_cents,
    amount / 100.0 AS amount_yuan,
    card_uid,
    created_at
FROM transactions
ORDER BY id DESC;

-- 所有金额应该是合理的分值（例如：11100 而不是 111）
```
